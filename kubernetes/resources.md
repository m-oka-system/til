## Kubernetes リソース一覧

| リソース名                               | 役割（なぜ必要か）                                                                                 |
| ---------------------------------------- | -------------------------------------------------------------------------------------------------- |
| Pod                                      | 実際のコンテナが動く最小単位。アプリ本体を動かすために必須。                                       |
| ReplicaSet                               | 同じ仕様の Pod を指定数だけ維持する管理者。障害で Pod が落ちても自動で補充し、稼働数を一定に保つ。 |
| Deployment                               | Pod の更新やスケールを自動管理する仕組み。安定したリリースとロールバックのために使う。             |
| StatefulSet                              | Pod に固定 ID とストレージを割り当てる管理方式。順序や状態が大事なワークロードで使う。             |
| DaemonSet                                | 各ノードに 1 つずつ Pod を配置する仕組み。ノード単位の監視やエージェント導入に使う。               |
| Job / CronJob                            | 1 回きり or 定期実行のバッチ処理を管理する仕組み。バッチやスケジュール処理のために使う。           |
| Service                                  | Pod 群に安定したアクセス先を提供する仮想 IP。Pod の入れ替えを意識せず通信するために必要。          |
| Ingress                                  | HTTP(S) の外部公開ルールをまとめるリソース。L7 ルーティングで複数サービスを 1 つの入口にまとめる。 |
| ConfigMap                                | 設定ファイルや環境変数を管理する仕組み。アプリの設定をコードから切り離すために使う。               |
| Secret                                   | パスワードや API キーなど機密情報を管理。暗号化された形で安全に渡すために必要。                    |
| PersistentVolume / PersistentVolumeClaim | 永続ストレージを宣言的に扱う仕組み。Pod が消えてもデータを残すために使う。                         |
