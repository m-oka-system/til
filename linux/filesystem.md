## ファイルシステムの利用状況を表示

```bash
df
-T　ファイルシステムのタイプを表示する
-h　容量に単位をつけて読みやすい形で表示する（kb=1024バイト）
-H　容量に単位をつけて読みやすい形で表示する（kb=1000バイト）
-i　iノードの利用状況を表示する
```

## ファイルやディレクトリごとの使用量を表示

```bash
du <file or directory>
du /var
-a　ディレクトリとファイルの両方を表示する
-l　リンクも含めて集計する
-c　すべての容量の合計を表示する
-s　指定したディレクトリの合計のみを表示する（サブディレクトリを省略）
-h　サイズに応じて読みやすい単位で表示する
-k　容量をキロバイト単位で表示する
-m　容量をメガバイト単位で表示する
-s　指定したファイルやディレクトリのみの合計を表示する
-S　サブディレクトリを含めずに集計する
```

## ファイルシステムをチェックする

```bash
※必ず対象のファイルシステムをアンマウントしてから実行すること

# fsck
fsck -t /dev/sdb2
-t　ファイルシステムの種類を指定する
-a　自動的に修復を実行する
-r　対話的に修復を実行する
-A　/etc/fstabに記述されている全ファイルシステムに対して実行する
-N　実際には実行せずに何が行われているのかを表示する

# e2fsck (ext2, ext3, ext4向け）
e2fsck /dev/sdb2
-p　すべての不良ブロックを自動的に修復する
-y　問い合わせにすべてYesと答える
-n　問い合わせにすべてNoと答える

```

## ダミーファイルの作成

```bash
## dd
sudo dd if=/dev/zero of=1G.dummy bs=1M count=1000
sudo dd if=/dev/zero of=100M.dummy bs=1M count=100

# fallocate
fallocate -l 1G 1G.dymmy
fallocate -l 100M 100M.dymmy
```

## シンボリックリンク

```bash
# シンボリックリンクを作成
ln -s <元ファイル名(リンク先)> <リンク名>
ln -s /Volumes/homes/azureuser DS218j

#　シンボリックリンクを削除
unlink <リンク名>
```
